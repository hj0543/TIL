
## 1. 캡슐화 (Encapsulation)

**"데이터와 기능을 하나로 묶고, 외부로부터 보호하기"**
객체의 필드(데이터)와 메서드(기능)를 하나로 묶고, 실제 구현 내용을 외부에 감추는 것을 말합니다.

* **정보 은닉:** `private` 같은 접근 제어자를 사용해 외부에서 데이터를 직접 변경하지 못하게 막고, 지정된 메서드(Getter/Setter)를 통해서만 접근하도록 제한합니다.
* **장점:** 객체의 내부 구조가 바뀌어도 외부 코드에 영향을 주지 않아 유지보수가 쉬워집니다.

## 2. 상속 (Inheritance)

**"부모의 유산을 물려받아 재사용하기"**
이미 정의된 상위 클래스(부모 클래스)의 특징을 하위 클래스(자식 클래스)가 물려받는 것입니다.

* **코드 재사용:** 공통된 기능을 부모 클래스에 한 번만 작성하면 자식 클래스에서 그대로 사용할 수 있습니다.
* **확장성:** 부모의 기능을 그대로 쓰면서도 자식 클래스만의 새로운 기능을 추가할 수 있습니다.

## 3. 추상화 (Abstraction)

**"중요한 특징만 뽑아내고 복잡한 건 숨기기"**
객체의 복잡한 내부 구현보다는 **'무엇을 하는지'**에 집중하여 공통적인 속성과 기능을 추출하는 과정입니다.

* **인터페이스와 추상 클래스:** 예를 들어 '자동차'라는 추상적인 개념을 만들 때, "가속한다", "브레이크를 밟는다"라는 핵심 기능(메서드)만 정의하고 구체적인 엔진 동작 방식은 각 자동차 모델(구현체)에서 처리하게 합니다.
* **장점:** 사용자는 내부 원리를 몰라도 인터페이스만 알면 객체를 사용할 수 있습니다.

## 4. 다형성 (Polymorphism)

**"하나의 모습이 여러 가지 형태로 변하기"**
하나의 변수나 메서드가 상황에 따라 다른 방식으로 동작하는 성질입니다.

* **오버라이딩(Overriding):** 부모 클래스의 메서드를 자식 클래스에서 용도에 맞게 재정의하는 것. (예: '소리내기' 메서드가 강아지에서는 '멍멍', 고양이에서는 '야옹'으로 동작)
* **오버로딩(Overloading):** 같은 이름의 메서드라도 매개변수의 타입이나 개수를 다르게 하여 여러 개 정의하는 것.
* **장점:** 코드의 유연성을 극대화하며, 부모 타입의 참조 변수로 여러 자식 객체를 관리할 수 있어 확장이 용이합니다.

### 한눈에 요약하기

| 특징 | 핵심 키워드 | 주요 목적 |
| --- | --- | --- |
| **캡슐화** | 클래스, 접근 제어자 | 정보 은닉 및 보호 |
| **상속** | `extends`, 부모-자식 | 코드 재사용 및 계층 구조 |
| **추상화** | `interface`, `abstract` | 복잡성 제거 및 모델링 |
| **다형성** | 오버라이딩, 오버로딩 | 유연성 및 확장성 |

이 4대 원칙은 서로 긴밀하게 연결되어 있어, 이를 잘 활용하면 깨끗하고 견고한 코드를 작성할 수 있습니다. 혹시 이 개념들 중 특정 언어(Java나 Python 등)의 예시 코드로 더 자세히 알고 싶은 부분이 있으신가요?
